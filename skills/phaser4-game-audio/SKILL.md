# Phaser 4 音频设计

读取 docs/project-profile.yaml、docs/GDD.md、docs/TDD.md、docs/control-plane.md；资源进入制作后读取 docs/asset-license-register.md，已有性能计划时再读取 qa-plan。音频方向、生成方式、授权、预算或发布资格需取舍时，向总控提交决策包，等待其完成决策门禁。

1. 按核心反馈、UI、环境、音乐和可访问性定义需求。
2. 在资源登记中记录来源、授权或生成记录、循环点、格式、时长、响度、大小、用途和发布资格。
3. 设计静音、分组、首次交互播放、暂停恢复和后台切换；在目标容器和浏览器验证。
4. 依据包体、内存和加载预算选择策略，标记未授权、不可循环、过大或不合渠道限制的资源。
5. 在控制面追加一条音频范围、证据或阻断项。

向玩法提供触发时机、分组和资源标识，向测试提供功能与预算项，向发布同步授权状态。
