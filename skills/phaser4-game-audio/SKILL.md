# Phaser 4 音频设计

音频方向、生成方式、授权、性能预算或发布资格需要人工选择，或需求发生变化时，先调用 $grilling；确认共同理解前不得生成或接入受影响音频。

## 执行流程

1. 读取 GDD、TDD、平台矩阵和资源登记，按核心反馈、UI、环境、音乐和可访问性定义音频需求。
2. 为每项音频记录来源、授权/生成记录、循环点、格式、时长、响度、文件大小、用途和发布资格，并登记到 asset-license-register.md。
3. 设计静音、音量分组、首次用户交互后播放、暂停恢复和后台切换行为；在目标容器及浏览器中验证。
4. 根据移动端包体、内存与加载预算选择压缩和加载策略。不要为了音质未验证地突破预算。
5. 将未获授权、不可循环、过大或不符合渠道限制的资源标为阻断或待替换。

## 交接标准

向玩法开发提供触发时机、音频分组和资源标识；向测试性能提供功能和预算测试项；向发布合规同步授权状态。更新资源登记、qa-plan.md 和 handoff-status.md。
