# Phaser 4 技术架构

建立可维护、可测试、可构建的边界，并显式处理小游戏、iOS、Google Play 的差异。

## 输入与边界

快速通道仅读取任务直接相关的代码、配置和测试；标准、发布通道读取 docs/project-profile.yaml、docs/GDD.md、docs/TDD.md、docs/control-plane.md，渠道进入候选后才读取平台矩阵。技术路线、平台适配、依赖或质量指标需要取舍时，向总控提交简短决策包，等待其完成决策门禁。

- 用 Phaser 4、TypeScript、Vite 实现 Web 核心；新项目优先采用官方当前的 create-phaser-game 流程。
- 用 Capacitor 承载 iOS 与 Android，将原生插件封装在平台适配层。
- 小游戏必须单独验证；可选商业能力须有显式接口和已批准开关。

## 执行与交接

1. 在 TDD 记录版本、模块职责、场景与状态、输入、资源、存档、错误处理、构建命令及风险。
2. 分离核心规则、表现、平台和第三方服务；为触控、安全区、暂停恢复、后台切换和断网定义验证策略。
3. 完成最小骨架和生产构建后，标准、发布通道在控制面追加一条证据或阻断项；快速通道以构建和测试结果作为交付证据。

交给玩法开发模块入口、数据流和测试入口；交给测试提供构建命令、版本和日志定位方式。新增或修改的类、函数、实体定义均添加解释设计边界的简体中文注释。
