# 效果图拆分为 Phaser 单图

把效果图中的可复用视觉元素转换成可直接加载、定位、打包和验证的独立资源。效果图不是最终资源，裁出矩形截图也不等于完成拆分。

## 输入与清单

1. 读取 GDD、TDD 和效果图，列出角色、部件、道具、地形、背景、UI、特效及状态变体；为每项标注用途、目标像素尺寸、缩放倍率、透明需求、锚点、九宫格或动画帧要求。
2. 区分应独立交互或复用的元素与仅属于完整背景的内容。不要拆出没有运行时用途的装饰碎片。
3. 遮挡、透视变形、光照烘焙或背景污染导致源图信息不足时，重绘或重新生成该元素；不得用带有其他元素残影、缺口或错误轮廓的裁片填补。

## 拆分与导出

1. 沿真实轮廓分离元素，移除背景、文字标注、选框、阴影污染和相邻元素；保留业务要求的自身阴影与发光。
2. 为被遮挡元素补全运行时会显露的区域。成组动画帧使用一致画布、基线、朝向和锚点，避免播放抖动。
3. 默认输出透明 PNG；仅对确定不需要透明的完整背景使用 JPEG 或 WebP。不要仅为“整齐”强制拉伸，也不要把非正方形资源填充成正方形。
4. 裁掉无效透明区域，同时为滤波、旋转或发光保留足够边距。进入图集的资源使用至少 2 px 挤出或等效防溢色设置，避免线性采样串色。
5. 使用稳定的小写短横线文件名；同一状态或动画采用连续、可排序的命名。按 TDD 输出独立文件或图集，并提供可直接使用的 Phaser 纹理键、帧名和默认锚点。

## 必须交付

- 独立图片或图集图片及对应 atlas JSON；不得只交效果图、切图坐标或编辑器工程文件。
- 每个资源的来源效果图、用途、像素尺寸、格式、透明方式、默认锚点、纹理键/帧名和文件路径记录。
- 对比预览：在棋盘格与深浅两种底色上展示逐图结果，便于发现白边、黑边、残留背景、截断和空白过大。
- Phaser 验证证据：实际通过 `this.load.image`、`this.load.atlas` 或对应加载方式载入，在目标缩放、相机和渲染设置下显示正确。

## 验收门槛

逐项确认后才能标记“可交付”：

- 画面完整：无误裁、残影、非预期背景、边缘缺口、透明杂点和可见接缝。
- 规格正确：像素尺寸、方向、状态、帧序、锚点、九宫格边界和命名与 TDD 一致。
- 渲染可靠：浏览器控制台无加载错误；纹理键与帧名可寻址；线性采样、缩放和旋转时无明显溢色或抖动。
- 性能合格：无重复纹理和无效大画布；图集尺寸、纹理数量、显存与包体符合已批准预算，并在至少一台目标设备验证。
- 可追溯：来源、生成或编辑记录、授权、责任人及发布资格已经写入资源登记。

任何一项失败都将该资源标为“待修复”，记录复现截图或运行步骤；不要用“接入时再处理”代替拆分验收。
