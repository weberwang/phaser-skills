# Phaser 4 美术接入

让视觉资源服务体验，并满足移动端性能和上架权属要求。

读取 docs/project-profile.yaml、docs/GDD.md、docs/TDD.md、docs/control-plane.md；资源进入制作后读取 docs/asset-license-register.md，已有性能计划时再读取 qa-plan。视觉方向、生成式资源、授权、预算或发布资格需取舍时，向总控提交决策包，等待其完成决策门禁。

1. 确定尺寸、分辨率、锚点、帧率、图层、九宫格、安全区与本地化要求。
2. 在资源登记中记录来源、授权或生成记录、用途、发布资格、责任人与优化状态；占位资源标明替换负责人和质量门。
3. 按 TDD 接入图集、动画和 UI，避免重复纹理、无界尺寸和无效透明区域，并在目标设备验证。
4. 在控制面追加资源范围、证据或阻断项。

向玩法提供规格和路径约定，向测试提供预算和验证证据，向发布提供权属清单。
