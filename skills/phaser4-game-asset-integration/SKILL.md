# Phaser 4 美术接入

## 职责

让视觉资源既服务游戏体验，也满足移动端性能和上架权属要求。可以协助生成可发布资源，但必须保留来源、提示词或授权、修改记录和发布资格。

视觉方向、生成式资源使用、授权风险、资源预算或发布资格需要人工选择，或需求发生变化时，先调用 $grilling；确认共同理解前不得生成或接入受影响资源。

## 执行流程

1. 读取 GDD、TDD、目标渠道和性能计划，确定尺寸、分辨率、锚点、帧率、图层、九宫格、安全区和本地化要求。
2. 将每个资源登记到 asset-license-register.md，标明来源、授权/生成记录、用途、发布可用状态、责任人和优化状态。
3. 临时占位资源必须标记替换负责人和最迟替换质量门；未经批准不得混入发布候选包。
4. 按 TDD 接入图集、动画和 UI 资源，避免重复纹理、无界尺寸和不必要透明区域；在目标设备验证加载、内存和清晰度。
5. 需要使用生成式图像时，先确认风格、商标、人物和渠道规则，再生成并记录可追溯信息。

## 交接标准

向玩法开发提供资源规格和路径约定，向测试性能提供资源预算与验证证据，向发布合规提供完整权属清单。更新资源登记和 handoff-status.md。
