# Phaser 4 美术接入

让视觉资源服务体验，并满足移动端性能和上架权属要求。

快速通道仅读取任务直接相关的资源、代码、规格和验证；标准、发布通道读取 docs/project-profile.yaml、docs/GDD.md、docs/TDD.md、docs/control-plane.md，资源进入制作后读取 docs/asset-license-register.md，已有性能计划时再读取 qa-plan。视觉方向、生成式资源、授权、预算或发布资格需取舍时，向总控提交决策包，等待其完成决策门禁。

1. 确定尺寸、分辨率、锚点、帧率、图层、九宫格、安全区与本地化要求。
2. 输入是包含多个元素的效果图、概念图或界面稿时，执行“效果图拆分为 Phaser 单图”任务，并完整读取 [references/effect-image-splitting.md](references/effect-image-splitting.md)。效果图只作为视觉来源，不得直接冒充可接入资源。
3. 在资源登记中记录来源、授权或生成记录、用途、规格与锚点、交付路径与纹理键、发布资格和 Phaser 验证；占位资源标明替换负责人和质量门。
4. 按 TDD 接入单图、图集、动画和 UI，避免重复纹理、无界尺寸、无效透明区域与采样溢色，并在目标设备验证。
5. 标准、发布通道在控制面追加资源范围、证据或阻断项；快速通道仅在产生未决决策或阻断项时追加。

向玩法提供规格、锚点、纹理键和路径约定，向测试提供预算、逐图检查结果与 Phaser 加载证据，向发布提供权属清单。拆分产物未通过逐图视觉检查和 Phaser 运行验证时，不得标为可交付。
