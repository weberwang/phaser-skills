# Phaser 4 发布与合规

## 工作边界

准备可提审交付物，不代表实际渠道提交、审核或上架已经完成。商店账号、签名密钥、令牌和个人数据只应存放在受控的外部密钥系统，绝不提交到仓库或项目文档。

渠道范围、合规风险接受度、隐私与数据处理、商店资料、提审豁免或发布放行需要人工决定，或这些需求发生变化时，先调用 $grilling；确认共同理解前不得生成受影响候选包或更改发布结论。

## 执行流程

1. 读取 project-profile.yaml、platform-matrix.md、asset-license-register.md、qa-plan.md、release-checklist.md 和 decision-log.md。
2. 按小游戏、iOS、Google Play 分别核对打包方式、版本号、图标与截图、描述、隐私、年龄分级、广告/内购披露、数据收集、账号删除和渠道特有要求。
3. 对“其他”渠道先新增独立矩阵行，完成技术适配、资料、合规和测试评审后再纳入候选范围。
4. 只从可复现的提交和构建命令生成候选包，并使候选包版本与测试证据一一对应。
5. 发现未授权资源、未确认数据处理、未经批准的商业能力、缺失渠道资料或测试空白时，标为 G3 阻断并请求人工决定。

## 交接标准

更新平台矩阵、发布检查清单、决策日志和交接状态。向人工提供每个渠道的候选包位置、完成项、阻断项、已知风险和建议结论；未收到明确放行前，不宣称可以提交或已经上架。
