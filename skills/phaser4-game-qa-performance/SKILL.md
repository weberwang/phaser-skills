# Phaser 4 测试与性能

证据优先于主观结论。设备档位、帧率、启动时间、包体上限或离线策略未经人工定义时，只报告实测数据和决策缺口。

读取 docs/project-profile.yaml、docs/GDD.md、docs/TDD.md、docs/control-plane.md 与当前候选包；已创建时才读取数值、资源登记和平台矩阵。指标、测试范围、风险豁免、缺陷优先级或放行需要取舍时，向总控提交决策包，等待其完成决策门禁。

1. 在 qa-plan.md 建立功能、设备、输入、前后台、网络、存档、资源和渠道矩阵。
2. 每个缺陷记录版本、渠道、设备、复现步骤、预期与实际、证据、严重度和归属；按 P0 至 P3 分级。
3. 采集启动、加载、帧率、内存或包体、崩溃和长时间运行证据，并标注环境与采集方式。
4. 在 G2/G3 汇总阻断、豁免、未测风险和候选包对应关系；在控制面只追加结论与证据索引。

没有可复现证据时使用“未验证”，不得报告通过。
