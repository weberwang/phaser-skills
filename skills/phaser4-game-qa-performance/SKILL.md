# Phaser 4 测试与性能

## 工作原则

从垂直切片开始测试，证据优先于主观结论。设备档位、帧率、启动时间、包体上限或离线策略没有人工定义时，只能报告实际数据和决策缺口，不能判定“达标”。

质量指标、测试范围、风险豁免、缺陷优先级或发布放行需要人工取舍，或其需求发生变化时，先调用 $grilling；确认共同理解前不改变测试基线或通过结论。

## 执行流程

1. 读取 GDD、TDD、数值、平台矩阵和当前候选包，建立功能、设备、输入、前后台、网络、存档、资源和渠道测试矩阵。
2. 对每个缺陷记录版本、渠道、设备、复现步骤、实际/预期结果、证据、严重度和归属；按 P0 至 P3 使用 qa-plan.md 的定义。
3. 采集冷启动、加载、帧率、内存/包体、崩溃和长时间运行证据，标注测试环境与采集方式。
4. 对触控、屏幕尺寸、安全区、暂停恢复、断网、资源加载失败和可选能力逐项验证。
5. 在 G2 和 G3 输出阻断项、豁免项、未测风险与候选包对应关系，交由人工判断是否放行。

## 交接标准

更新 qa-plan.md、release-checklist.md 和 handoff-status.md。只报告“通过”或“失败”时必须附带可复现证据；没有证据时使用“未验证”。
