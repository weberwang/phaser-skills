# Worktree 工作区隔离

Worktree 只用于隔离文件修改和 Git 状态，不作为质量门、角色交接、阶段交付物或自动收敛机制。

- 多个并发写入者：每个写入者使用独立 worktree 和分支，并明确文件或模块归属；不得让两个写入者修改同一文件。
- 保护已有脏工作区：先检查当前分支、提交和 `git status`。新任务不依赖未提交改动时，从已提交基线创建新 worktree，不 stash、reset、移动或覆盖用户改动。
- 依赖未提交改动：继续在原工作区谨慎处理，不创建无法看到这些改动的隔离 worktree。
- 无需隔离：单个写入者、顺序任务和只读任务直接使用当前工作区。

任务完成后在对应 worktree 运行必要检查，并按普通 Git 流程提交和集成。删除 worktree 或分支前重新确认工作目录干净、提交已保留且目标明确；不自动合并、清理或强制删除。
