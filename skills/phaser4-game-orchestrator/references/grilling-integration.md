# 拷问集成

总控在需要人工选择前执行：

1. 读取现有文档、实现和证据，分离事实与取舍。
2. 只有存在必须人工取舍的未知项时才调用 $grilling；相关低风险默认项可组成一个决策包，高影响、相互依赖或冲突项逐项询问，并给出影响、推荐项和理由。
3. 未获结论前仅冻结直接受影响范围；无依赖工作继续。确认后才改对应需求、实现、资源或发布状态。
4. 在 docs/control-plane.md 记录日期、触发原因、事实、问题、结论、影响和下一步；将细节写入受影响交付物。

功能增删、优先级或验收变化、渠道或能力开关变化、技术或数值路线变化、视觉/资源变化、质量指标变化和发布豁免，均按需求修改处理。影响既有质量门时标为待复核。
