# 游戏实现与切图集成

进入 G1 或 G2 的游戏实现时读取本文件。实现不是单一编码动作，而是按验收增量完成代码、资源、验证和可运行构建的闭环。

## 必须任务链

1. **冻结实现增量**：从 GDD 选择已批准且有可观察验收条件的功能，记录玩家行为、依赖、排除项和测试点；未批准需求不得实现。
2. **建立技术骨架**：按 TDD 完成场景、状态、输入、资源、配置、存档与平台边界，保证干净环境可以安装、启动和生产构建。
3. **建立资源依赖清单**：玩法在编码场景表现前列出每项图片的用途、状态、目标尺寸、纹理键/帧名、锚点、九宫格或动画要求，并查询资源登记。
4. **分流图片资源**：已有有效拆分记录的资源直接复用；合成效果图进入美术接入的“登记查重 → 审查拆分方案 → 拆图 → 逐图检查 → Phaser 验证 → 更新登记”流程；缺失资源可使用显式占位，但必须登记替换负责人和最迟质量门。
5. **实现核心规则**：玩法并行实现与表现解耦的规则、状态、场景流转、输入、UI 行为、数值配置、生命周期及已批准的数据能力，不等待切图完成才开始无依赖编码。
6. **接入正式资源**：只使用审查通过且登记有效的文件、纹理键、帧名和锚点替换占位资源；不得从效果图自行裁切、复制已有拆分项或使用未登记文件。
7. **完成集成验证**：验证核心循环、触控、重开、暂停恢复、前后台切换、资源失败、离线和适用的存档异常；逐场景检查缩放、旋转、动画、九宫格、安全区、采样溢色和资源释放。
8. **完成测试与构建**：运行核心规则测试、功能验收和生产构建，在至少一个目标渠道或设备记录启动、加载、帧率、内存、纹理数量及包体证据。
9. **提交实现证据**：记录实现提交或变更文件、运行/测试/构建命令、验收结果、正式资源清单、剩余限制和阻断项；更新 GDD 状态与控制面索引。

## 资源与实现回路

- 切图可以与纯规则实现并行，但正式场景接入必须等待对应资源通过拆分前审查、逐图检查和 Phaser 验证。
- 玩法发现边缘、尺寸、锚点、帧序或九宫格问题时，将运行证据交回美术；美术修复并更新登记后，玩法重新接入，测试只复核受影响路径。
- 来源版本、拆分范围、规格、纹理键或锚点变化时，资源重新查重和审查；同时把依赖该资源的实现增量标为待复核。
- G2 前清除所有未获明确豁免的占位资源。效果图、未验收裁片、重复拆分文件和已取代版本不得进入候选构建。

## 完成定义

实现增量只有在代码可运行、功能验收通过、正式资源可追溯、生产构建可复现且阻断缺陷已处置时才算完成。文档、效果图、资源清单或角色声明不能单独作为实现完成证据。
