# 项目交付物规范

默认初始化只生成核心交付物；其余文档在对应角色真正开工前用 --include 创建。不要读取、创建或维护当前阶段不需要的文件。

| 类型 | 文件 | 负责人 | 创建时机 |
| --- | --- | --- | --- |
| 核心 | docs/project-profile.yaml | 制作策划 | 初始化 |
| 核心 | docs/GDD.md | 制作策划 | 初始化 |
| 核心 | docs/TDD.md | 技术架构 | 初始化 |
| 核心 | docs/control-plane.md | 总控；各角色仅追加状态 | 初始化 |
| 可选 | docs/balance.md | 数值设计 | 数值进入实施 |
| 可选 | docs/asset-license-register.md | 美术接入、音频 | 资源进入制作、效果图拆分或接入 |
| 可选 | docs/qa-plan.md | 测试性能 | G1 前 |
| 可选 | docs/platform-matrix.md | 发布合规 | 渠道进入候选 |
| 可选 | docs/release-checklist.md | 发布合规 | G3 前 |

标准、发布通道的控制面只保留当前质量门、待决策、阻断项、有效变更和角色交接摘要；快速通道仅在产生未决决策或阻断项时使用。详细规则、证据与清单写在所属交付物中。质量门汇合后删除已写入基线且不再影响后续工作的行，不保留问答全文。状态使用：未开始、进行中、待人工决策、待验收、已验收、阻断、废弃。

缺少信息时在控制面新增一条待决策，写明影响和最迟决策点。不得记录密钥、账号、个人数据、未脱敏日志或受限合同全文。
