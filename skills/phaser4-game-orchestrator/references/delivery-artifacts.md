# 项目交接物规范

总控和角色 skill 均以此文件定义的文档作为交接契约。初始化脚本会生成最小模板；可以补充项目内容，但不要删除字段含义。

| 文件 | 负责人 | 最小内容 | 主要消费者 |
| --- | --- | --- | --- |
| docs/project-profile.yaml | 制作策划 | 项目名、渠道、能力开关、状态、未决指标 | 全部角色 |
| docs/GDD.md | 制作策划 | 核心循环、玩家旅程、范围、验收条件 | 玩法、数值、测试 |
| docs/TDD.md | 技术架构 | 模块边界、构建、存档、平台适配、风险 | 玩法、测试、发布 |
| docs/balance.md | 数值设计 | 参数、公式、目标、模拟与验证记录 | 玩法、测试 |
| docs/asset-license-register.md | 美术接入与音频 | 资源、来源、授权、用途、优化状态 | 发布、测试 |
| docs/qa-plan.md | 测试性能 | 测试矩阵、缺陷分级、设备和性能证据 | 全部角色 |
| docs/platform-matrix.md | 发布合规 | 渠道差异、包、商店资料、合规与状态 | 总控、测试 |
| docs/G0-人工决策请求.md | 总控 | 立项质量门的选项、影响、推荐与批准结论 | 人工决策者 |
| docs/grilling-log.md | 制作策划与触发角色 | 拷问触发、逐项问答、共同理解与影响范围 | 全部角色 |
| docs/worktree-plan.md | 总控与并行角色 | 分支、worktree、范围、检查、提交、收敛状态与证据 | 总控、全部角色 |
| docs/release-checklist.md | 发布合规 | 候选版本逐项提审核验 | 人工决策者 |
| docs/decision-log.md | 制作策划 | 待决策、选择、证据、决策人、日期 | 全部角色 |
| docs/handoff-status.md | 每位角色 | 输入、输出、状态、阻塞、证据链接 | 总控 |

## 文档更新规则

- 在开始会改变范围、技术、资源、体验或发布条件的工作前，先检查决策日志中是否已有批准结论。
- 使用可追溯链接、提交号、版本号、构建命令或测试记录替代“已完成”这类无证据描述。
- 缺少信息时在 decision-log.md 新增一条待人工决定事项，并说明不决定的影响和最迟决策点。
- 任何待人工决定事项和需求修改都必须先关联 grilling-log.md 中已确认的拷问记录。
- 不在项目文档中写入密钥、账号、用户数据、未脱敏日志或受限合同全文。

## 角色状态值

使用 未开始、进行中、待人工决策、待验收、已验收、阻断、废弃 之一。只有下游消费者确认输入可用时，才能将交接状态标为已验收。
