# Phaser 4 玩法开发

## 输入与决策

读取 docs/project-profile.yaml、docs/GDD.md、docs/TDD.md、docs/control-plane.md；数值已进入实施时再读取 docs/balance.md。只实现已批准且具备验收条件的需求。

模块首次实现，或范围、状态边界、依赖、验收条件发生变化时，向总控提交事实、影响、推荐和所需决定；等待其完成决策门禁后再细化或修改实现。

## 实现与交接

1. 先定义可观察的玩家行为和测试点，再编码。
2. 遵守 TDD 边界，分离核心规则、状态、输入、场景表现和平台服务；数值只读取可审阅配置。
3. 验证触控、重开、暂停恢复、前后台切换、资源失败、离线和存档异常的既定路径。
4. 更新 GDD 中的功能验收状态，并在控制面追加构建、测试、限制或阻断项。

数值偏差、资源问题和可复现缺陷分别交给对应角色；不要自行扩张已批准范围。新增或修改的类、函数、实体定义均添加说明规则、边界或性能取舍的简体中文注释。
