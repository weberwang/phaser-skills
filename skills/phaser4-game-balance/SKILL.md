# Phaser 4 数值设计

## 职责

将体验目标转换为可解释、可调整和可验证的模型。不要用未记录的主观调参替代目标和证据。

数值目标、难度取舍、经济风险、商业化影响或可接受偏差需要人工决定，或已批准模型发生需求修改时，先调用 $grilling；确认共同理解前不改变数值基线。

## 执行流程

1. 读取 GDD、项目能力开关和当前玩法实现，定义玩家节奏、成功率、资源来源与消耗、失败成本和保底机制。
2. 在 balance.md 记录每个参数的含义、单位、默认值、来源、影响范围、公式和验证方法。
3. 将运行时参数集中在可审阅的配置中；玩法代码只能消费配置，不能复制公式或硬编码重复值。
4. 为高风险曲线建立表格、脚本或可复现计算过程，并同时记录模拟假设和实际试玩结果。
5. 发现概率、付费、奖励或难度会影响合规、未成年人保护或商业能力时，提交人工决策请求并通知发布合规角色。

## 交接标准

交给玩法开发的每个模型必须有参数表、边界行为和验收样例；交给测试性能的每个模型必须有关键场景和预期结果。更新 balance.md、decision-log.md 和 handoff-status.md。
